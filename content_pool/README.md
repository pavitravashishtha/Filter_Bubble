# Content Pool Datasets ‚Äî Filter Bubble Simulation

Generated by `generate_content_pools.py` (seed=42). Three files, one per platform,
matching Section 1 of the design document.

## 1. `imdb_content_pool.csv` ‚Äî REAL DATA (1,000 rows)

Source: a scraped IMDb Top-1000 Movies dataset (title, year, genre, rating,
Metascore, votes, gross ‚Äî see `source` column: `real` for all 1,000 base rows).
This is genuine IMDb data, not synthetic.

| Column | Derivation |
|---|---|
| `title`, `released_year`, `release_era`, `runtime_minutes` | Direct from source data |
| `genre_list`, `genre_vector` | Parsed from source `Genre` field; `genre_vector` is a 21-dim multi-hot (see `imdb_genre_index.json`) |
| `imdb_rating_raw`, `imdb_rating_normalized` | Raw 1‚Äì10 IMDb rating; normalized via the design doc's `(raw-1)/9` formula (Section 3.1) |
| `meta_score`, `critical_consensus` | Raw Metascore (0‚Äì100); `critical_consensus` = Metascore/100 |
| `no_of_votes`, `mainstream_appeal` | Raw vote count; `mainstream_appeal` = log-normalized votes, 0‚Äì1 |
| `gross_usd` | Raw box office gross (0 where missing) |
| `position_in_belief_space` | **Not derivable from real data** ‚Äî drawn from Normal(5.0, 2.0) per Section 1.3, same as every platform |
| `emotional_valence` | Heuristic: mean of a genre‚Üívalence weight table (e.g. Horror=0.85, Comedy=0.3) + noise. **This is a documented assumption**, not sourced from IMDb ‚Äî flag it as such in your methods section |
| `engagement_potential` | Blend of `mainstream_appeal` and log-normalized gross + noise |
| `quality_signal` | Average of `critical_consensus` and `imdb_rating_normalized` |
| `novelty`, `birth_timestep` | All 1.0 / 0 at pool creation (simulation applies decay from here) |

**Not included ‚Äî generate at simulation init, not in the dataset file:**
`latent_features` (20‚Äì30 dim, random init per SGD scheme), `ideological_loading_vector`,
`item_bias` (init 0.0). These are learned/initialization-time vectors per Section 3.1,
not properties of the movie itself.

**Scope note:** the design doc's target range is 5,000‚Äì10,000 items; the real
dataset caps at 1,000 unique titles. `IMDB_TARGET_POOL_SIZE` in the script can pad
the pool with additional rows sampled from the *empirical* genre/rating/vote
distributions of the real 1,000 (via `synthesize_additional_movies`) ‚Äî these are
clearly flagged `source=synthetic_augmented` and must **not** be presented as real
IMDb titles in your writeup. Recommended: report results on the real-only 1,000-item
pool, and mention the padding option only if you need to test pool-size sensitivity.

## 2. `reddit_content_pool.csv` ‚Äî SYNTHETIC (2,500 rows)

Generated per Section 1.2/1.6. All fields synthetic by design (the doc specifies
synthetic generation for Reddit, not a real dataset).

| Column | Generation method |
|---|---|
| `community_id` | Uniform random over 20 communities |
| `content_type` | 70% post / 30% comment |
| `upvotes`, `downvotes` | Lognormal net-karma draw (skewed, mimics real vote distributions) split by a noisy upvote ratio centered at 0.85 |
| `timestamp` / `birth_timestep` | Uniform over first 50 timesteps (initial pool "seeding" window) |
| `cross_post_origin` | 5% of items flagged as cross-posted from a random other community (-1 = none) |
| `position_in_belief_space` | Normal(5.0, 2.0), clamped 0‚Äì10 |
| `emotional_valence`, `quality_signal` | Beta(2,2) / Normal(0.5, 0.2), clamped |
| `engagement_potential` | Blend of normalized net-karma and randomness |

## 3. `youtube_content_pool.csv` ‚Äî SYNTHETIC (4,000 rows)

Generated per Section 1.2/1.6.

| Column | Generation method |
|---|---|
| `topic_cluster` | Uniform random over 15 clusters |
| `emotional_valence` | Each of the 15 clusters gets its own valence center (Uniform 0.2‚Äì0.85), items drawn around their cluster's center ‚Äî this seeds the "some clusters run hotter" structure the RL rabbit-hole mechanism is meant to exploit |
| `thumbnail_appeal` | Beta(2,2) |
| `creator_authority` | Normal(0.5, 0.2), clamped |
| `completion_rate_baseline` | Fixed 0.5 for all items, per design doc ("initialised at 0.5") |
| `predicted_watch_time` | Blend of baseline, thumbnail appeal, and randomness |
| `position_in_belief_space` | Normal(5.0, 2.0), clamped 0‚Äì10 |

## Honest limitations to state in your paper

1. Only IMDB uses real data; Reddit and YouTube are fully synthetic by design (matching the design doc's own specification, not a shortcut taken here).
2. `emotional_valence`, `engagement_potential`, and `quality_signal` for the real IMDB rows are **heuristic derivations**, not IMDb-sourced fields ‚Äî IMDb has no native "emotional valence" of a movie. Report the genre-weight table in your methods appendix.
3. `position_in_belief_space` is synthetic for all three platforms (no real dataset encodes political/ideological position) ‚Äî this is consistent with the design doc, which specifies the same Normal(5,2) init for every platform regardless of data source.
4. The Reddit/YouTube vote and engagement distributions are statistically plausible (lognormal/beta shapes matching known engagement skew) but not calibrated against real Reddit/YouTube statistics. If you want to strengthen this later, calibrating the lognormal parameters against publicly reported subreddit vote distributions would be a lightweight improvement.
